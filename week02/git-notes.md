# Git Notes

# Local vs remote 
- A local repository is part of the project that’s saved on my computer.
- A remote repository is the part that's stored online(on GitHub).

# Repo, commit, branch
- A repository is basically the main project folder that Git keeps track of.
- A commit is saving/updating the changes you made so Git remembers everything
- A branch lets you work on something new without messing up the project.

# staging area
- The staging area is where you pick which changes you want to include before making a commit.

# push vs pull
- Push is sending your commits from your computer to the remote repository.
- Pull means getting the newest updates from the remote repository onto your computer.

# what .gitignore does
- A .gitignore file tells Git which files it should ignore or not keep up with. 
- This is useful for files like dependencies or other files that don’t need to be in the repo.


# what a README is for
- A README file explains what the project is about and how it works.
- It's basically just a description of everything.
- It usually gives information or instructions for anyone looking at the project.
